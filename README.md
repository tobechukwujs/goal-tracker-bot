## âš™ï¸ Local Installation & Setup

### 1. Clone the Repository
```bash
git clone [https://github.com/tobechukwujs/goal-tracker-bot.git](https://github.com/tobechukwujs/goal-tracker-bot.git)
cd goal-tracker-bot

```

### 2. Install Dependencies

```bash
npm install

```

### 3. Environment Configuration

Create a `.env` file in the root directory and add your secrets:

```env
TELEGRAM_TOKEN=your_telegram_bot_token_here
GEMINI_API_KEY=your_gemini_api_key_here
DATABASE_URL=postgresql://user:password@host:port/database
TZ=Africa/Lagos  # Sets the bot's timezone

```

### 4. Database Setup

Run the following SQL queries in your Supabase/PostgreSQL SQL Editor to set up the required tables:

```sql
-- Create Users Table
CREATE TABLE IF NOT EXISTS users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    telegram_id BIGINT UNIQUE,
    chat_id BIGINT UNIQUE,
    first_name TEXT,
    username TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create Goals Table
CREATE TABLE IF NOT EXISTS goals (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    goal_text TEXT NOT NULL,
    priority INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create Daily Activities Table
CREATE TABLE daily_activities (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    user_id BIGINT REFERENCES users(id) ON DELETE CASCADE,
    content TEXT,
    activity_date DATE DEFAULT CURRENT_DATE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    PRIMARY KEY (user_id, activity_date)
);

```

### 5. Run Locally

To start the bot in development mode (auto-restarts on save):

```bash
npm run dev

```

To start normally:

```bash
npm start

```

## â˜ï¸ Deployment (Render)

1. **Push to GitHub:** Ensure your code is on a private GitHub repository.
2. **Create Web Service:** Go to [Render](https://render.com), create a new **Web Service**, and connect your repo.
3. **Settings:**
* **Build Command:** `npm install`
* **Start Command:** `node bot.js`


4. **Environment Variables:** Add `TELEGRAM_TOKEN`, `GEMINI_API_KEY`, `DATABASE_URL`, and `TZ` in the Render dashboard.
5. **Keep-Alive:** Use a service like **UptimeRobot** to ping your Render URL every 5 minutes to prevent the free tier from sleeping.
